二、软件使用

使用方法：直接在命令行输入命令.

进入 cmipwork 所在目录，运行命令格式如下：

  • -f：从文件(file)中读取问题或高级启动信息.

  • -s：设置(set)参数，参数格式为键值对，即 [key] [value]. 可输入的键(key)有：

键 含义
cmip/timeTolerance 时间限制(Time limit)，单位：秒
print/solOutput 解的输出文件路径(File path for output solution)
cmip/gapTolerance Gap limit
cmip/nodeTolerance Node limit

运行命令示例如下：

  • ./bin/cmip -f XXX.mps

    读取 XXX.mps，并求解.

  • ./bin/cmip -f XXX.mps -s cmip/timeTolerance 3600 cmip/gapTolerance 0.01

    读取 XXX.mps；最长求解时间为 3600 秒；求解 gap 为 0.01.

  • ./bin/cmip -f XXX.mps -s print/solOutput ./solOutput.txt

    读取 XXX.mps；将求出的解存储在 ./solOutput.txt 文件中.

三、软件测试

执行 make test [TEST=XXX] [TIME=time] 进行测试，后可加相关参数：

  • TEST=XXX ：XXX 表示在 check/testsets/ 目录下的 XXX.test 文件，存放待测试的算例，默认为 testeasy. testeasy 对应于 check/testsets 目录下的 testeasy.test 与 testeasy.solu 文件(后者可以没有)，testeasy.test 存放每个算例的路径，testeasy.solu 存放算例的最优值.
  • TIME=time ：time 为时间上限，单位为秒，默认值为 7200.

四、接口参考

1、环境说明

需要包含 cmipwork/src/cmip/CMIP.h 文件，

2、模型输入

（1）首先利用CMIP* cmip = new CMIP(env) 来建立一个CMIP对象.

（2）利用 cmip->CMIPNewVariables(NVARS ,obj ,lb, ub, ctype, cname)来添加变量和目标函数.

  • NVARS - int,变量数量.

  • obj - double*, 目标函数系数.

  • lb - double*, 变量下界.

  • ub - double*, 变量上界.

  • ctype - CMIPVarType, 变量类型.

  • cname - string, 变量名称.

(3) 利用 cmip->CMIPAddConstraints(NCONS, nnz, lhs, rhs, rmatbeg, rmatind, rmatval, cname) 来添加约束.

  • NCONS - int, 约束数量.

  • nnz - int, 非零元数量.

  • lhs - double*, 约束左端项.

  • rhs - double*, 约束右端项.

  • rmatbeg - int*, 和rmatind以及rmatval一起定义所要添加的约束.

    rmatbeg用来定义非零元索引，
    第i条约束的列标被记录在rmatind[rmatbeg[i]]到rmatind[rmatbeg[i+1]-1]之间.
    第i条约束的系数被记录在rmatval[rmatbeg[i]]到rmatval[rmatbeg[i+1]-1]之间.

  • rmatind - int*, 见rmatbeg.

  • rmatval - double*, 见rmatbeg.

  • cname - string,约束名称.

(4) 其他工具

  • cmip->WriteModel(true, "origin.mps") - 输出.mps格式原始文件.

  • cmip->WriteModel(true, "origin.lp") - 输出.lp格式原始文件.

  • string cmip->origModel->name - 原始模型名称.

  • string cmip->origModel->filename - 原始文件名称.

  • string cmip->transModel->filename - 转化后模型名称.

  • string cmip->transModel->filename - 转化后文件名称.
